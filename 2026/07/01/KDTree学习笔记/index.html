小程序游戏性能优化
抖音小游戏首包与启动网络性能优化笔记项目背景某 Unity WebGL 抖音小游戏在抖音开发者工具性能检测中出现首包体积与网络性能问题： 首包大小超过平台限制，目标需控制在 30MB 以内。 FP 阶段存在 tt.request 平均耗时偏高问题。 构建产物主要体积集中在 webgl.data，资源治理优先级高于代码体积优化。 优化目标 降低 Unity WebGL 首包体积，满足抖音小游戏首包限制。 减少首屏阶段非必要 TTSDK 请求。 保持广告、侧边栏奖励等商业化功能可用，但改为按需触发。 建立可复用的资源扫描、导入配置与构建验证流程。 问题定位通过 Unity Build Report 与构建产物分析，确认主要问题包括： 大体积字体源文件被直接打入首包。 TMP 示例资源位于 Resources 目录，被 Unity 自动收集进包体。 可选 UI prefab、图片资源放在 Resources 下，导致未使用资源进入 webgl.data。 部分纹理、音频资源导入参数不适合小游戏首包场景。 启动期存在 TTSDK 初始化、侧边栏可用性检查、加载场景自动插屏等非必要...
