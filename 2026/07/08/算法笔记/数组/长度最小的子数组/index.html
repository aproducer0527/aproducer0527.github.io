长度最小的子数组 LeetCode209

发表于2026-07-08|更新于2026-07-08|算法笔记
|浏览量:

移除元素

问题描述

给定一个含有 n 个正整数的数组和一个正整数 target 。
找出该数组中满足其总和大于等于 target 的长度最小的 子数组 [numsl, numsl+1, …, numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0
对应LeetCode209

核心思路

维护一个滑动窗口[i,j],j不断向右扩展,sum保存当前窗口内元素和,当sum大于等于
target时 当前窗口满足条件，尝试缩小左边界i 继续找更短的答案

流程图示意

复杂度

时间复杂度：O(n)：虽然外层有一个 for 循环，内部还有一个 while 循环，看起来像嵌套循环，但时间复杂度不是 O(n²)。右指针 j 从数组开头移动到末尾最多移动 n 次，左指针 i 也只会不断向右移动每个元素最多被加入窗口一次也最多被移出窗口一次因此两个指针总共移动次数最多为 n+n=2n即为O(n)

空间复杂度：O(1)：没有创建额外数组、哈希表或其他数据结构
